# Nand Dairy API ‚Äî Complete Postman Testing Guide
Base URL: `https://nand-dairy-backend.onrender.com`

> **Setup**: After logging in, copy the `token` value. In Postman, go to **Authorization** tab ‚Üí **Bearer Token** ‚Üí paste token. Do this for all requests below.

---

## 1. Health Check
**GET** `/api/health`  
No auth needed. Verifies server + database are both up.

---

## 2. Authentication
**POST** `/api/auth/login`  
Body:
```json
{ "mobile_number": "9999999999", "password": "admin123" }
```
‚Üí Returns `token` and `user` object with `role`.

---

## 3. Users / Accountants *(Admin only)*

| Method | URL | Body |
|--------|-----|------|
| GET | `/api/users` | ‚Äî |
| GET | `/api/users/:id` | ‚Äî |
| POST | `/api/users` | `{ "name", "mobile_number", "password", "role" }` |
| PUT | `/api/users/:id` | `{ "name", "mobile_number", "role" }` or add `"password"` to reset it |
| PATCH | `/api/users/:id/toggle-active` | `{ "is_active": false }` |

**Valid roles:** `admin`, `milk-entry`, `fat-snf`, `report`

**Example POST body:**
```json
{
  "name": "Ramesh",
  "mobile_number": "8888888888",
  "password": "pass123",
  "role": "milk-entry"
}
```

## 5. Vehicles *(Admin only for POST/DELETE, all for GET)*

| Method | URL | Body |
|--------|-----|------|
| GET | `/api/vehicles` | ‚Äî |
| POST | `/api/vehicles` | `{ "vehicle_number": "RJ14XY1234", "samiti_id": 1 }` |
| DELETE | `/api/vehicles/:id` | ‚Äî |
| GET | `/api/vehicles/search/:vehicle_number` | ‚Äî |

**Example search:** `GET /api/vehicles/search/RJ14XY1234`  
‚Üí Returns `samiti_id`, `samiti_name`, `code_4digit`

## 6. Milk Entries *(milk-entry or admin role)*

| Method | URL | Notes |
|--------|-----|-------|
| POST | `/api/milk-entries` | Submit a collection |
| GET | `/api/milk-entries/today` | All today's entries |
| GET | `/api/milk-entries/today?shift=morning` | Filter by shift |

**POST body:**
```json
{
  "samiti_id": 1,
  "shift": "morning",
  "entry_date": "2026-04-05",
  "milk_quantity_liters": 50.5
}
```
> ‚öÝÔ∏è Duplicate same samiti+shift+date will return **409 Conflict** error.

---

## 7. Fat/SNF Entries *(fat-snf or admin role)*

| Method | URL | Notes |
|--------|-----|-------|
| GET | `/api/fat-snf/pending` | Queue of milk entries without fat/snf |
| POST | `/api/fat-snf` | Submit fat/snf for a milk entry |

**POST body:**
```json
{
  "milk_entry_id": 1,
  "fat_value": 6.5,
  "snf_value": 8.5,
  "rate_per_liter": 45.0
}
```
> **Validation:** fat must be 0‚Äì10, snf must be 0‚Äì15, rate must be ‚â• 0.  
> `total_amount` is auto-calculated: `milk_qty √ó rate_per_liter`

---

## 8. Reports *(admin or report role)*

| Method | URL | Query Params |
|--------|-----|-------------|
| GET | `/api/reports/daily` | `?date=2026-04-05` |
| GET | `/api/reports/bill` | `?samiti_id=1&from_date=2026-03-26&to_date=2026-04-05` |

**Daily response includes:** samiti name, total milk, avg fat, avg snf, total amount  
**Bill response includes:** date-by-date rows with milk, fat, snf, rate, amount

## Full Workflow Test Sequence

1. ‚úÖ Login as admin ‚Üí get token
2. ‚úÖ `POST /api/samitis` ‚Üí note down samiti `id`
3. ‚úÖ `POST /api/vehicles` ‚Üí link vehicle number to samiti
4. ‚úÖ `GET /api/vehicles/search/<vehicle_number>` ‚Üí confirm lookup
5. ‚úÖ `POST /api/users` ‚Üí create a milk-entry accountant
6. ‚úÖ Login as that milk-entry user ‚Üí get new token
7. ‚úÖ `POST /api/milk-entries` ‚Üí submit morning entry
8. ‚úÖ Login as fat-snf user ‚Üí get new token
9. ‚úÖ `GET /api/fat-snf/pending` ‚Üí confirm entry shows up
10. ‚úÖ `POST /api/fat-snf` ‚Üí submit fat/snf values
11. ‚úÖ Login as admin again
12. ‚úÖ `GET /api/reports/daily?date=today` ‚Üí see summary
13. ‚úÖ `GET /api/reports/bill?samiti_id=1&from_date=...&to_date=...` ‚Üí see bill
